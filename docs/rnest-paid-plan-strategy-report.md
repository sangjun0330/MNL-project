## 1. 결론 요약

RNest는 더 이상 단순한 "간호사 바디배터리 PWA"가 아닙니다. 현재 코드 기준 RNest는 아래 4개 축을 가진 간호사용 생활 운영 플랫폼입니다.

| 축 | 현재 상태 | 과금 적합도 | 역할 |
| --- | --- | --- | --- |
| 회복/컨디션 관리 | 매우 강함 | 매우 높음 | 핵심 구독 상품 |
| 현장 간호 툴 | 강함 | 높음 | 생산성 업셀 |
| 소셜/그룹/챌린지 | 중간~강함 | 중간 | 리텐션/바이럴 |
| 쇼핑/커머스 | 시스템은 강함, 카탈로그 성숙도는 낮음 | 낮음~중간 | 부가 매출/제휴 |

핵심 판단은 다음과 같습니다.

1. RNest의 유료화 중심은 반드시 `회복 인텔리전스 + 현장 시간절감 + AI 안전보조`여야 합니다.
2. `일정 입력`, `기록`, `기본 인사이트`, `기본 계산기`는 무료에 남겨야 사용자 습관과 신뢰가 형성됩니다.
3. 현재 Pro는 `AI 맞춤회복`과 `AI 약물·기구 안전 가이드`에 너무 과도하게 의존합니다. 지금 구조만으로는 구독 동기가 좁습니다.
4. 소셜과 쇼핑은 지금 당장 주 과금축으로 밀기보다, 무료 리텐션과 유료 전환 보조 장치로 써야 합니다.
5. 가장 빠른 매출 개선 포인트는 "새 기능을 많이 만드는 것"보다 `유료 가치 묶음 재정렬`과 `전환 타이밍 설계`입니다.

추천 방향은 아래와 같습니다.

- 단기: 현재 `Free / Pro / Credit Pack` 구조를 유지하되, Pro 가치 묶음을 크게 강화
- 중기: `Free / Plus / Pro` 3단 구조로 확장
- 장기: `개인 구독 + 그룹/학교/병동 단위 팀 플랜` 검토

## 2. 현재 RNest 서비스/기능/시스템 분석

### 2.1 핵심 사용자 서비스

현재 RNest의 사용자 표면은 `홈`, `일정`, `인사이트`, `툴`, `소셜`, `쇼핑`, `설정/구독`으로 구성됩니다.

#### A. 회복/컨디션 관리

- 교대근무 캘린더 입력
- 일별 건강 기록 입력
  - 수면
  - 낮잠
  - 스트레스
  - 활동량
  - 카페인
  - 기분
  - 생리/증상
  - 근무 이벤트 태그/메모
- Body/Mental battery 계산
- 홈 화면 오늘 컨디션 요약
- 인사이트 잠금 해제 구조
  - 최소 3일 기록 후 기본 인사이트 노출
- 인사이트 상세
  - 바이탈 요약
  - 주간 요약/트렌드
  - 에너지 소모 요인 분석
  - AI 맞춤회복 상세

판단:

- RNest의 본질적 경쟁력은 이 영역에 집중되어 있습니다.
- 특히 "교대근무 기반 회복 처방"은 일반 웰니스 앱과 차별화됩니다.

#### B. 현장 간호 툴

- 로컬 계산기
  - 펌프 변환
  - IVPB
  - 드립 계산
  - 희석/농도
  - 역산 검산
- 계산기 고급 모드 성격의 기능
  - 프리셋 저장
  - 더블체크
  - 처방 목표 비교
  - 최근 계산 기록
- AI 약물·기구 안전 가이드
  - 의약품/기구/상황 질문
  - 병동/ER/ICU 모드
  - 투여 전/중/이상반응 대응 맥락
  - SBAR, 환자 설명, 위험 신호, 중단 기준
  - 이미지 업로드 지원
  - 최근 검색 기록

판단:

- 계산기는 매우 강한 획득 기능입니다.
- AI 안전 가이드는 비용이 크지만, 지불 의사가 높은 고가치 기능입니다.
- 단, 계산기 내부에서 이미 `전문 모드`라는 표현을 쓰고 있어 향후 실제 유료 플랜 명칭과 충돌할 가능성이 큽니다.

#### C. 소셜

- 친구 연결 코드
- 친구 일정 공유
- 공통 OFF 탐색
- 그룹 생성/가입/초대
- 그룹 공지/활동 로그
- 그룹 멤버 건강 공유 기반 랭킹
- 그룹 챌린지
  - 리더보드형
  - 낮은 값 경쟁형
  - 그룹 목표형
  - 연속 달성형
- 실시간 새로고침/이벤트 센터

판단:

- 소셜은 RNest의 습관 유지 장치로 매우 유용합니다.
- 하지만 현재 코드 기준으로는 "유료 결제 이유"보다 "계속 쓰는 이유"에 가깝습니다.
- 소셜을 바로 강하게 과금하면 반감이 큽니다.

#### D. 쇼핑/커머스

- 회복 상태 기반 상품 추천
- 카탈로그/검색/카테고리/위시리스트/장바구니
- 주문/결제/리뷰/배송지/배송조회
- 파트너 제휴형 외부 판매
- 인앱 결제형 구조까지 이미 준비
- 배송 추적, 주문 이메일, 환불, 관리자 운영

판단:

- 시스템은 이미 전자상거래 수준으로 넓게 준비되어 있습니다.
- 그러나 기본 카탈로그는 아직 `제휴 준비중`, `가격 연동 예정` 상태가 많아 성숙도가 낮습니다.
- 즉, 지금은 "수익 인프라"는 있으나 "상품 경쟁력"은 아직 부족합니다.

### 2.2 현재 결제/과금 구조

현재 코드에서 확인되는 결제 구조는 다음과 같습니다.

| 구분 | 현재 구조 |
| --- | --- |
| Free | 통계형 인사이트 + 핵심 일정/건강 기록 |
| Pro | 월 14,900원, AI 맞춤회복 + AI 검색 기본 크레딧 10회/일 + 최근 검색 10개 저장 |
| Credit Pack | 10회 1,500원, AI 약물·도구 검색 추가 크레딧 |

현재 구조의 문제는 아래와 같습니다.

1. Pro 가치가 너무 좁습니다.
2. AI 검색 크레딧팩이 너무 저렴하면 구독 유인이 약해집니다.
3. 회복/인사이트/계산기/소셜이 대부분 무료인데, 실제로 돈 내게 만드는 기능은 AI 2개뿐입니다.
4. "매일 써야 하는 핵심 루프"와 "돈을 내야 하는 기능"의 연결이 아직 약합니다.

### 2.3 운영 시스템/인프라 분석

RNest는 사용자 기능보다 운영 시스템이 이미 더 넓게 깔려 있습니다. 이 점은 유료화에 매우 중요합니다. 왜냐하면 "무엇을 팔 수 있는가"는 화면보다 운영구조가 먼저 결정하기 때문입니다.

#### 현재 확인되는 운영 시스템

- 앱 프레임워크
  - Next.js App Router
  - React 19
  - PWA 설치 지원
- 인증/데이터
  - Supabase OAuth 로그인
  - 사용자 상태 스냅샷 저장
  - 상태 revision 보관
  - AI 결과 캐시 저장
- 결제/구독
  - TossPayments 구독/추가 크레딧 결제
  - 결제 승인/실패/웹훅
  - 환불 요청 및 관리자 검토
- AI 운영
  - AI 맞춤회복 결과 캐시
  - AI 약물·기구 가이드 사용량 차감
  - 일일 크레딧 초기화 개념
- 커머스 운영
  - 주문/취소/환불
  - 배송지 프로필
  - 배송 조회
  - 주문 메일 발송
- 소셜 운영
  - 그룹/이벤트/챌린지 진행률 동기화
  - 실시간 갱신
- 스케줄 잡
  - 배송 상태 동기화
  - 미결제 주문 정리
  - 소셜 챌린지 진행률 동기화
- 보안
  - CSP
  - HSTS
  - same-origin 요청 검사
  - 관리자 접근 분리
- 다국어
  - 한국어/영어 지원

#### 이 시스템이 의미하는 것

1. RNest는 이미 "실험 앱" 단계를 벗어났습니다.
2. 지금 필요한 것은 기능 추가보다 "어떤 기능을 어떤 가격 구조로 묶을지"에 대한 제품 전략입니다.
3. 특히 AI, 결제, 환불, 배송, 관리자 운영이 이미 들어가 있기 때문에 유료 플랜은 충분히 현실적인 단계입니다.

#### 운영 관점 핵심 판단

- `AI 맞춤회복`은 구독형 상품으로 적합
- `AI 안전 가이드`는 구독형 + 사용량형 혼합 과금이 적합
- `쇼핑`은 구독 본체보다 별도 매출 라인
- `소셜`은 구독보다는 리텐션과 그룹 플랜의 씨앗

#### 주의할 점

- 환불이 아직 관리자 수동 운영 성격이 강해, 유료 가입자 확대 시 운영부담이 커질 수 있습니다.
- AI 기능이 늘수록 원가와 의료적 책임 문구 관리가 더 중요해집니다.
- 쇼핑 카탈로그 성숙도가 낮은 상태에서 커머스를 과도하게 전면 배치하면 제품 포커스가 흐려질 수 있습니다.

## 3. RNest에서 무료에 남겨야 하는 것

무료에 남겨야 하는 기준은 다음 3가지입니다.

1. RNest의 핵심 습관을 만드는 기능
2. 사용자가 신뢰를 형성하는 기본 기능
3. 원가 부담이 낮고 바이럴/유입 기여가 큰 기능

### 무료 유지 권장

#### 필수 무료

- 로그인/계정 동기화
- 일정 캘린더 입력
- 기본 건강 기록 입력
- 생리 주기 설정
- 홈의 오늘 컨디션 기본 점수
- 기본 인사이트 잠금 해제 후 통계형 요약
- 기본 바이탈/트렌드/배터리 소모 요인
- 기본 로컬 계산기 전부
- 소셜 친구 연결/공통 OFF 탐색 기본 기능
- 쇼핑 탐색/찜/기본 추천 보기

#### 무료에 남겨야 하는 이유

- 일정 입력과 건강 기록을 막으면 RNest 자체가 성장하지 않습니다.
- 기본 계산기를 유료화하면 신뢰보다 반감이 먼저 생깁니다.
- 인사이트 일부를 무료로 보여줘야 "더 깊은 분석을 돈 내고 보겠다"가 성립합니다.

## 4. 유료 플랜에 넣어야 하는 것

유료 기능은 아래 기준으로 묶는 것이 맞습니다.

1. 개인화 깊이가 높다
2. 반복 사용 가치가 크다
3. 시간 절감이 크다
4. AI/외부 API 원가가 발생한다
5. 사용자가 "없으면 불편하다"를 느낀다

### 4.1 Pro에 즉시 넣어야 하는 것

현재 구조를 유지하면서 가장 빠르게 강화할 항목입니다.

#### 회복/인사이트 유료화

- AI 맞춤회복 전체
- 오늘 회복 처방 히스토리
- 14일/30일 회복 예측
- 교대근무별 비교 분석
- 수면 부채/카페인 패턴/야간 적응 리포트
- 위험 알림
  - 과도한 카페인
  - 수면 부족 누적
  - 야간 연속 근무
  - 생리/증상 악화 구간
- "다음 근무까지 회복 계획" 자동 생성
- What-if 시뮬레이션
  - 오늘 일찍 자면 내일 점수 변화
  - 카페인 컷오프 조정 시 변화
  - 근무 교체 시 영향

#### 현장 툴 유료화

- AI 약물·기구 안전 가이드
- AI 검색 기록 확장
- 고급 계산기 기능만 유료 전환
  - 프리셋 저장
  - 더블체크
  - 처방 목표 차이 추적
  - 인계용 텍스트 자동 요약
  - 개인 프리셋 클라우드 동기화

#### 왜 맞는가

- 기본 계산은 무료로 남기고, "실무 효율을 높이는 고급 기능"만 유료화해야 저항이 적습니다.
- AI 안전 가이드는 원가와 즉시가치가 모두 크므로 유료 정당성이 가장 높습니다.

### 4.2 무료에서 빼면 안 되는 것

유료 유도를 위해 아래를 빼는 것은 오히려 역효과가 큽니다.

- 일정 입력 자체
- 건강 기록 입력 자체
- 기본 점수 확인
- 기본 배터리 추세
- 로컬 계산 결과
- 계정 동기화/복구
- 기본 소셜 연결

## 5. 현재 구조에서 가장 큰 문제점

### 문제 1. Pro가 "AI만 있는 플랜"처럼 보임

사용자는 "회복 앱"에 돈을 내는 것이 아니라, "매일 시간을 줄여주고 실수를 줄여주는 시스템"에 돈을 냅니다.  
지금 RNest Pro는 가치가 강하지만 너무 좁습니다.

### 문제 2. 계산기 `전문 모드` 네이밍이 혼란을 줌

현재 계산기 내부의 `basic / pro`는 진짜 과금 플랜이 아니라 UI 모드입니다.  
이 상태로 실제 유료 플랜을 확장하면 사용자는 혼란을 느낍니다.

권장:

- 계산기 내부 `전문 모드`를 `고급 모드`로 리네이밍
- 실제 결제 플랜은 `Plus` 또는 `Pro`로 분리

### 문제 3. 개인화 설정은 엔진에 있지만 UI 노출이 약함

코드상 배터리 엔진은 이미 아래 개인화 변수를 받습니다.

- chronotype
- caffeineSensitivity

하지만 사용자에게 이 강력한 개인화 입력이 거의 전면 노출되지 않습니다.

이 말은 곧:

- 새 엔진을 새로 만드는 것보다
- 기존 개인화 모델을 제품 기능으로 꺼내는 것이
- 더 빠르고 싸며 유료가치도 높다는 뜻입니다.

### 문제 4. 쇼핑은 과금 보조축이지 메인 구독 이유가 아님

쇼핑 시스템은 잘 준비돼 있지만, 현재 카탈로그 성숙도와 제휴 상태를 보면 "구독 이유"로 쓰기엔 아직 이릅니다.

권장:

- 쇼핑 자체를 유료 잠금으로 만들지 않음
- 대신 Pro 혜택으로 연결
  - 개인화 추천 강화
  - 제휴 할인
  - 회복 키트 큐레이션

### 문제 5. 소셜은 과금보다 리텐션 레버

소셜은 사용자가 앱에 남게 만드는 힘이 큽니다.  
하지만 지금 단계에서 소셜 핵심 기능을 강하게 잠그면 사용자 성장이 꺾입니다.

권장:

- 소셜 기본은 무료
- 그룹 리더용 고급 기능만 나중에 유료

## 7. 유료 결제를 유도하려면 추가로 확장해야 할 기능

### 7.1 가장 먼저 확장해야 할 기능

#### 1. 회복 플래너

사용자가 가장 돈을 낼 가능성이 높은 기능입니다.

- 다음 근무까지 남은 시간 기준 루틴
- 지금 자면/지금 카페인 마시면/낮잠 자면 점수 변화 예측
- 연속 야간 근무 회복 프로토콜
- 오프데이 회복 계획
- 생리/PMS 포함 개인 루틴

이유:

- RNest의 핵심 정체성과 완전히 일치
- 매일 반복 사용 가능
- AI/비AI 혼합 구성 가능

#### 2. 고급 계산기 프로덕티비티

- 병동/개인 프리셋 클라우드 저장
- 최근 계산 히스토리 검색
- 교대 인계용 한 줄 복사 템플릿
- 더블체크 완료 표시
- 자주 쓰는 약물 세팅 북마크

이유:

- 원가가 낮고, 실무 효율 가치가 높음
- "무료 계산기"와 차별화가 쉬움

#### 3. AI 안전 가이드 고도화

- 최근 기록 정리
- 자주 조회한 약물 즐겨찾기
- 이미지 OCR 정확도 개선
- 병동별 템플릿
- "중단/보고 기준만 빠르게 보기" 요약 모드

이유:

- 고위험/고가치 영역이라 지불 전환이 빠름
- 다만 의료책임 리스크 관리가 중요

### 7.2 두 번째로 확장할 기능

#### 4. 알림/자동화

- 수면 시작 권장 시점 알림
- 카페인 컷오프 알림
- 야간 근무 전 준비 알림
- 회복 점수 급락 시 경고

이유:

- 매일 앱을 다시 열게 하는 강력한 리텐션 장치
- 구독 지속률 개선에 직접적

#### 5. 개인화 설정 UI

- 아침형/야행성 성향
- 카페인 민감도
- 나이트 적응 난이도
- 생리 민감도

이유:

- 이미 엔진이 일부 준비되어 있어 구현 효율이 좋음
- 유료 플랜 정확도 체감이 확 올라감

### 7.3 이후 확장할 기능

#### 6. 그룹 프리미엄

- 그룹장 전용 분석 리포트
- 그룹 챌린지 커스텀 생성
- 그룹별 회복 랭킹 상세
- 학교/병동/스터디 그룹 플랜

#### 7. 커머스 프리미엄 혜택

- Pro 전용 제휴 할인
- 회복 루틴 번들
- 교대근무 키트 정기 큐레이션

주의:

- 쇼핑은 구독 본체가 아니라 혜택 패키지로 다루는 것이 더 좋습니다.

## 8. 전환 포인트 설계

유저는 기능 설명보다 "필요한 순간"에 더 잘 결제합니다.

### RNest에서 가장 강한 결제 타이밍

1. 건강 기록 3일 달성 직후

- 야간 연속근무 감지 시 "이번 주 회복 계획 받기" 업셀

## 11. 최종 권장안

가장 현실적인 전략은 아래입니다.

### 제품 원칙

- 무료는 "써보는 앱"이 아니라 "계속 쓰는 기본 시스템"으로 만든다
- 유료는 "AI만 쓰는 플랜"이 아니라 "회복 + 실무 효율 + 안전" 패키지로 만든다
- 소셜은 과금보다 리텐션을 맡긴다
- 쇼핑은 주 구독 이유가 아니라 혜택과 제휴 매출로 연결한다

### 최종 권장 분리

#### 무료 유지

- 일정
- 기록
- 기본 점수
- 기본 인사이트
- 기본 계산기
- 기본 소셜
- 쇼핑 탐색

#### 유료 전환

- AI 맞춤회복
- 회복 히스토리
- 회복 예측
- 고급 계산기 기능
- AI 안전 가이드
- 고급 알림
- 깊은 개인화

### 가장 먼저 만들어야 할 3가지

1. 회복 플래너
2. 개인화 설정 UI
3. 고급 계산기 생산성 기능

이 3개가 붙으면 RNest Pro는 "AI 몇 번 쓰는 플랜"이 아니라 "교대근무 간호사의 시간과 컨디션을 실제로 관리해 주는 플랜"으로 바뀝니다.

## 12. 한 줄 제안

RNest는 `무료로 습관을 만들고, 유료로 회복 성과와 실무 효율을 사게 만드는 구조`로 가야 합니다.  
즉, `기본 기록/기본 계산/기본 통계는 무료`, `개인화 회복 계획 + AI 안전 가이드 + 고급 생산성 기능은 유료`가 가장 맞는 방향입니다.
